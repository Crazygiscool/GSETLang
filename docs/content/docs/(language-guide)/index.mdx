## Syntax Conventions

### Blocks

GSET uses braces `{}` for code blocks, similar to JavaScript, Java, and Go:

```go
if condition {
    # code here
}
```

### Statements

Statements are typically terminated by newlines. Semicolons are optional:

```go
x = 1
y = 2  # both valid
```

### Comments

Use `#` for single-line comments:

```go
function main() {
    # This is a comment
    print("Hello")  # inline comment
}
```

## Comparison with Other Languages

| GSET Feature | Python | JavaScript | Java |
|-------------|--------|------------|------|
| Variables | `x = 1` | `let x = 1` | `int x = 1` |
| Functions | `def fn()` | `function fn()` | `void fn()` |
| Classes | `class Foo` | `class Foo` | `class Foo` |
| Lists | `[1, 2, 3]` | `[1, 2, 3]` | `new int[]` |

<Callout type="info">
  GSET's syntax is intentionally flexible to feel natural to developers from different backgrounds.
</Callout>
